# InRem 사용자 여정 지도 (User Journey Map)

> 문서 버전: v1.0 · 최종 업데이트: 2026-05-19  
> 대상: 1차 페르소나 P1(김재훈, 47세 1인 가구) 중심  
> 관련 문서: `personas.md`, `PRD.md`

---

## 0. 여정 개요

InRem의 사용자 여정은 **5단계 라이프사이클**로 구성된다.

```
[1] 인지(Awareness)
   └─→ [2] 가입·온보딩(Onboarding)
         └─→ [3] 습관 형성(Habit Loop - Stage 1: Guardian Pulse)
               └─→ [4] 깊은 사용 전환(Stage 2: Heritage Box / End Note)
                     └─→ [5] 사후 집행(Stage 3: Legacy Sync, 보호자 주도)
```

각 단계는 다음 차원에서 분석한다.
- **Action** (행동): 사용자가 실제로 무엇을 하는가
- **Touchpoint** (접점): 어떤 화면/채널에서 일어나는가
- **Thought** (생각): 머릿속에 무엇이 떠오르는가
- **Emotion** (감정): 어떤 기분을 느끼는가 (😟😐🙂😄)
- **Pain Point** (페인): 무엇이 막거나 불편한가
- **Opportunity** (기회): 우리가 어떻게 도울 수 있는가

## 1. Stage 0 — 인지 (Awareness)

| 차원 | 내용 |
| --- | --- |
| Action | 동료가 1인 가구 돌연사 뉴스 공유 → 구글에서 "1인 가구 안부 앱" 검색 → 앱스토어 추천 발견 |
| Touchpoint | 뉴스 기사, 카톡 대화, 구글 검색, 앱스토어 |
| Thought | "남 일 같지 않다. 내가 그렇게 되면 누가 알지?" |
| Emotion | 😟 불안 → 😐 정보 탐색 모드 |
| Pain | 비슷한 앱이 너무 많거나(SOS·헬스케어) 너무 가벼움(카카오 안부). 신뢰할 만한 브랜드가 없음 |
| Opportunity | **ASO/SEO**: "1인 가구 안부", "디지털 유산", "사후 정리" 키워드 선점. **신뢰 시그널**: 후기·언론·전문기관 협력 노출 |

## 2. Stage 1 — 가입·온보딩 (Onboarding, Day 0)

| 차원 | 내용 |
| --- | --- |
| Action | 앱 설치 → 이메일/비밀번호 가입 → 권한 안내(알림) → 안심 시간(수면 시간) 설정 → 보호자 1명 초대(선택, 스킵 가능) |
| Touchpoint | `SignupScreen` → `OnboardingFlow` → `SettingsScreen`(quiet hours) → `GuardianInvite` 모달 |
| Thought | "권한이 과하면 끌까…" / "보호자 초대는 부담스러운데 나중에 해도 되나?" |
| Emotion | 🙂 기대 + 😐 망설임 |
| Pain | 권한 요청이 과하거나, 보호자 등록을 강제하면 이탈. "감시당한다"는 느낌이 들면 즉시 삭제 |
| Opportunity | - 권한 요청 사유를 **구체 시나리오**로 설명 ("8시간 이상 무활동 시 알림")<br>- 보호자 초대 **스킵 허용**, 7일 후 재권유<br>- 안심 시간을 **첫 화면에서 손쉽게** 설정 (오탐 예방 효과를 미리 보여주기) |

### KPI
- 가입→홈화면 도달률: 목표 ≥ 80%
- 7일 내 보호자 1명 이상 등록률: 목표 ≥ 40%

## 3. Stage 2 — 습관 형성 (Habit Loop, Day 1–30)

이 단계는 **Guardian Pulse**의 일상 사용 루프다.

### 3-A. 정상 활동 흐름 (Happy Path)

| 시점 | Action | Touchpoint | Emotion |
| --- | --- | --- | --- |
| 매일 아침 | 앱 열어 타이머 확인 | `HomeScreen` 원형 타이머 | 🙂 안심 |
| 수동 체크인 | 가끔 SwipeToConfirm으로 안부 확인 | `HomeScreen` 스와이프 | 😄 통제감 |
| 무의식 활동 | 그냥 앱 화면만 열어도 heartbeat 전송 | 백그라운드 `useAppStateHeartbeat` | 😐 인지 못함 (의도) |

### 3-B. 무활동 감지 흐름 (Detection Path)

```
[T=0] 마지막 활동
   ↓ (X시간 경과, 사용자가 설정한 임계치)
[T=X] Soft Check-in 푸시
   "잘 지내시나요?" + [잘 있어요] 버튼
   ↓ 응답 O → 타이머 리셋, 정상 루프 복귀 (Happy)
   ↓ 응답 X (Y분 경과)
[T=X+Y] Hard Check-in 에스컬레이션
   → 보호자에게 FCM 멀티캐스트 알림
   → FCM 실패 시 이메일 발송
```

| 차원 | 내용 |
| --- | --- |
| Action | 푸시 받음 → 잠금화면에서 그대로 액션 응답 / 무시 시 보호자가 전화 |
| Touchpoint | FCM 푸시, 시스템 알림, (실패 시) 이메일 |
| Thought | "와 이거 작동하는구나" / "괜한 알림 아닌가?" |
| Emotion | 😄 신뢰 또는 😟 알림 피로 |
| Pain | 오탐(false positive)이 많으면 알림 끄거나 앱 삭제. 응답 버튼이 멀거나 클릭이 어려우면 무시 |
| Opportunity | - 안심 시간(quiet hours) 적용으로 새벽 알림 차단<br>- **잠금화면 1탭 응답** (Notification Action)<br>- 오탐 감소 위한 **활동 패턴 학습** (Phase 5) |

### KPI
- D7 리텐션: 목표 ≥ 50%
- 무활동 알림 응답률: 목표 ≥ 85%
- 오탐 비율: 목표 ≤ 5%

## 4. Stage 3 — Stage 2 전환: 깊은 사용 (Day 30–90)

사용자가 "안부 체크"에 익숙해진 시점에, **Heritage Box / End Note**로 자연스럽게 유도한다.

| 차원 | 내용 |
| --- | --- |
| Action | 홈 화면 카드: "오늘은 디지털 유산 1개 정리해 보세요" → Heritage Box 진입 → SNS 계정 1개 입력(이름/타입/사후 처리 의사) |
| Touchpoint | `HomeScreen` 추천 카드, `HeritageBoxScreen`, `AssetFormScreen` |
| Thought | "갑자기? 부담스럽긴 한데… 1개만 해볼까" |
| Emotion | 😐 망설임 → 🙂 정리되는 느낌 |
| Pain | "한 번에 다 정리해야 한다"는 압박감. 입력 폼이 길거나 의무 필드가 많으면 이탈 |
| Opportunity | - **1번에 1항목** 원칙 (Atomic Task)<br>- **사후 처리 의사**는 4지선다 (삭제/기념/승계/비공개)로 단순화<br>- 분기별 "정리 리포트" 발송으로 보람감 제공<br>- 가족공유 옵션은 **유료**로 묶어 결제 전환 트리거 |

### KPI
- Stage 2 진입률 (Heritage Box 1항목 작성): 목표 ≥ 25%
- Stage 2 진입자 중 유료 전환률: 목표 ≥ 15%

## 5. Stage 4 — 사후 집행 (Execution, Trigger Event)

이 단계는 **사용자가 사망한 후 보호자가 주도**하는 여정이다.

| 시점 | Action | Touchpoint | 책임자 |
| --- | --- | --- | --- |
| T=0 | Hard Check-in 무응답 + 가족 사망 확인 | 보호자 앱, 외부 공증 | 보호자 |
| T+1d | Executor가 InRem 사망 신고 모드 활성화 | `ExecutorScreen` | Executor |
| T+3d | Heritage Box 인벤토리 기반 체크리스트 자동 생성 | `LegacySyncScreen` | Executor |
| T+7d | 전문가 매칭 (상조/세무/법무) | 외부 제휴 | InRem 중개 |
| T+30d | 디지털 계정 정리 진행률 리포트 | 이메일/리포트 | Executor |

| Emotion | 😢 슬픔 → 😐 책임감 → 🙂 정리 완료 후 안도 |
| Pain | 사망 증빙 절차의 신뢰성, 권한 위임 절차의 법적 안정성 |
| Opportunity | - **사망 확인은 다중 검증**(보호자 2명 + 사망진단서 업로드)<br>- **단계별 안내 SLA**(48시간 내 응답)<br>- 추모관(Eternal Link) **옵트인 기본 OFF** |

## 6. 페르소나 P2 (보호자) 여정 요약

```
[알림 수신] → [피보호자에게 전화] → [응답 O: 안도]
                                    ↘ [응답 X: 119 또는 직접 방문]
                                                      ↘ [Executor 모드 진입]
```

핵심 요구사항:
- 보호자 앱은 **알림 수신·대시보드 확인** 중심으로 단순.
- 피보호자의 평소 패턴(주간 활동 그래프) 시각화로 "잘 지내고 있구나" 안심.

## 8. 핵심 모먼트 오브 트루스 (Moments of Truth)

서비스 성패를 가르는 5가지 결정적 순간.

1. **온보딩 권한 요청**: 거부하면 영구 이탈. 사유를 명확히.
2. **첫 Soft Check-in 응답**: 오탐이면 즉시 알림 끔. 임계치 보수적으로 시작.
3. **30일차 Stage 2 유도**: "충분히 신뢰가 쌓인 후"에만 권유. 너무 빠르면 거부감.
4. **첫 결제 전환**: 가족공유 / 리포트 / 저장 용량 — 어디서 끊을지가 ARR 결정.
5. **사망 확인 절차**: 한 번의 오탐도 치명적. 다중 검증 + 인간 검토 필수.
