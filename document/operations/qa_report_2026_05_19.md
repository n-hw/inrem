## 1. 발견된 버그 (수정 완료)

### 🐞 Bug #1 — `<SafeAreaProvider>` 누락 → React 트리 crash
- **증상**: Expo Web 로 띄우면 페이지가 완전히 빈 화면. 콘솔에 `pageerror: No safe area value available. Make sure you are rendering <SafeAreaProvider> at the top of your app.`
- **원인**: `App.tsx` 최상위에 `react-native-safe-area-context` 의 `SafeAreaProvider` 가 없음. iOS/Android 에서도 `useSafeAreaInsets` (BottomTabBar) 와 `SafeAreaView` (ScreenLayout) 가 사용 중이라 다른 환경에서도 잠재적 위험.
- **수정**: `App.tsx` 의 `<GestureHandlerRootView>` 아래에 `<SafeAreaProvider>` 추가.

### 🐞 Bug #2 — `expo-secure-store` 가 Web 에서 crash
- **증상**: 회원가입 직후 `pageerror: n.default.deleteValueWithKeyAsync is not a function`. 토큰 저장 실패 → 인증 상태로 진입 못함.
- **원인**: `expo-secure-store` 는 iOS Keychain / Android Keystore 전용. Web 빌드에서 실행되면 함수 자체가 없음.
- **수정**: `front/src/api/client.ts` 의 `tokenStorage` 를 platform-aware 로 변경. Web 은 `localStorage`, 네이티브는 `SecureStore`. 주석에 "Web 은 localStorage 라 실제 'secure' 하지 않음 — production target 은 native" 명시.

### 🐞 Bug #3 — `Alert.alert` 가 RN Web 에서 invisible
- **증상**: 잘못된 로그인 / 비밀번호 불일치 등 사용자 입력 에러 시 화면에 아무 피드백이 표시되지 않음. 사용자는 무슨 일이 일어났는지 알 수 없음.
- **원인**: `Alert.alert` 는 RN Web 에서 `window.alert` 폴백을 시도하지만 dismiss/blocking 차이로 실질적으로 안 보이거나 즉시 닫히는 경우가 많음.
- **수정**: `LoginScreen` / `SignupScreen` 에 inline 에러 배너 (`errorBanner` 상태 + `<View>`) 추가. Alert 의존 제거. Web/Native 모두 동일하게 노출.
- **검증**: 시각 QA 시나리오 2 에서 잘못된 로그인 → 빨간색 배너 "로그인이 필요해요. 다시 로그인해 주세요." 노출 확인 (`visual-02-login-error.png`).

- **장시간 활동 시나리오**: 12시간 이상 비활동 → 푸시 → 무응답 → 이메일 fallback → 보호자 알림 전체 루프 — 시간 기반이라 별도 통합 테스트 또는 시뮬레이션 필요.

- **부하 / 동시성**: 단일 인스턴스 가정 한정. Redis 기반 rate-limiter 로의 교체 + 부하 테스트는 출시 직전 단계.
