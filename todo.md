## 🔮 Phase 5: 고급 안전 기능 (다음 단계)

### 1. AI 보이스 콜 (Automated Care)
- **목표**: 푸시/SMS에도 응답이 없을 경우, 시스템이 직접 전화를 걸어 안부 확인.
- **기술**: VAPI.ai 또는 Clam API (음성 인식 및 합성).
- **흐름**:
    1. 하드 체크인 타임아웃 발생.
    2. 시스템이 VoIP/PSTN 전화 발신.
    3. AI 에이전트: "안녕하세요, 잘 지내시나요?" 질문.
    4. STT(음성 텍스트 변환)로 응답 분석.
    5. "도와줘" 또는 무응답 시 -> 긴급 연락처로 즉시 에스컬레이션.

### 2. 생활 패턴 분석 (Bio-rhythm)
- **목표**: 완전한 비활동 상태가 오기 *전에* 이상 징후를 감지.
- **기술**: `ActivitySignal` 데이터 기반 머신러닝 (scikit-learn).
- **기능**:
    - 사용자의 기상/수면 패턴 학습.
    - 평소보다 훨씬 늦게 기상하는 등 패턴 이탈 시 조기 알림.
    - 활동 추세 대시보드 시각화.

### 3. 긴급 구조 서비스 통합
- **목표**: 119 또는 경찰서 시스템과의 직접 연결 (추후 확장).
- **기능**:
    - 피보호자 화면에 "원터치" 긴급 호출 버튼 제공.
    - GPS 위치 및 의료 정보 자동 공유 시스템.

    - 익명 지원 그룹.
    - 복지사 연동 Q&A 섹션.

## 📝 기술적 부채 및 최적화

- **테스트 커버리지**: 단위 테스트 커버리지를 80% 이상으로 확보.
- **Redis 도입**:
    - 인메모리(변수) 기반 디바운싱 로직을 Redis로 교체하여 서버 다중화 대비.
    - Celery/Task 큐 도입 시 브로커로 활용 (현재는 APScheduler 사용 중).
- **웹소켓 실시간 업데이트**:
    - 가디언 대시보드에서 폴링(Polling) 대신 Socket.IO를 사용하여 실시간 상태 반영.
